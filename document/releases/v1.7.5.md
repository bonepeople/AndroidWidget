## Release Note

This release adds two integration-facing capabilities — a lifecycle-aware global event bus and name-based serial coroutine execution — and upgrades Kotlin and kotlinx-coroutines. Existing integrations are not affected unless you adopt the new APIs or have strict dependency version alignment requirements.

### New: `AppEvent` — Lifecycle-Aware Global Event Bus

A new in-process event bus is available under `com.bonepeople.android.widget.util.event`.

- **`AppEvent.register(owner, minState, action)`** — Binds a subscription to a `LifecycleOwner` (e.g. `Activity`, `Fragment`). The subscription is automatically cancelled on destroy; no manual unregister is required.
- **`AppEvent.post(event)`** — Posts an event from a coroutine context.
- **`AppEvent.postAsync(event)`** — Posts an event from a non-coroutine context (callbacks, click listeners, etc.) via `CoroutinesHolder.default`.

Define events by implementing the `AppEventData` marker interface. Prefer typed `data class` events for maintainability; use `CommonEvent(name, data)` for generic signal-only or payload-based events.

**Behavior highlights:**

- **Non-sticky delivery** — Subscribers do not receive events posted before registration. Events emitted while a subscriber is inactive (lifecycle below `minState`) are also not replayed on resume.
- **Lifecycle-gated receiving** — Posting is independent of subscriber lifecycle; receiving is gated by `minState` (default: `STARTED`).
- **In-process only** — Event objects are passed by reference within the same process.
- **Overflow policy** — Uses `DROP_OLDEST`; under extreme load, older buffered events may be dropped.

See [AppEvent documentation](../features/AppEvent/README.md) for usage examples.

### Enhanced: `CoroutinesHolder` — Name-Based Serial Execution

Three new APIs allow coroutine-level mutual exclusion by task name:

- **`runSerial(name, block)`** — Runs a suspend block serially within the caller's coroutine context.
- **`launchSerial(name, scope, context, start, block)`** — Launches a serial coroutine; defaults to `CoroutinesHolder.default` (`Dispatchers.Default`).
- **`asyncSerial(name, scope, context, start, block)`** — Same as `launchSerial`, but returns a `Deferred` for tasks with a result.

Tasks sharing the same `name` never run concurrently; tasks with different names are independent. `name` must not be blank.

**Important:** The underlying `Mutex` is not reentrant. Do not call `runSerial` with the same `name` from within a block already running under that name (including inside `launchSerial` / `asyncSerial`), or a deadlock will occur.

See [CoroutinesHolder documentation](../features/CoroutinesHolder/README.md) for details.

### Dependency and Compatibility Updates

| Component | Previous | Current |
|-----------|----------|---------|
| Kotlin Gradle Plugin | 1.8.22 | 1.9.25 |
| kotlinx-coroutines-core | 1.6.4 | 1.8.1 |
| kotlinx-coroutines-android | 1.6.4 | 1.8.1 |

Both coroutines artifacts are exposed as `api` dependencies, so consuming projects inherit this version. If your app pins an older coroutines version, resolve potential version conflicts in your dependency resolution strategy. No changes to minimum SDK, target SDK, compile SDK, Java compatibility, or Android Gradle Plugin requirements.

### Upgrade Recommendation

Recommended for projects that need lifecycle-safe cross-component event delivery or lightweight serial task coordination without introducing a separate EventBus library. No migration is required for existing integrations.

## Changelog

- Added `AppEvent` lifecycle-aware global event bus with `register`, `post`, and `postAsync` APIs, backed by `MutableSharedFlow` with `DROP_OLDEST` overflow handling.
- Added `AppEventData` marker interface for typed event definitions.
- Added `CommonEvent` generic event type with `name` and optional `data` payload for signal-only or lightweight event scenarios.
- Added `CoroutinesHolder.runSerial`, `launchSerial`, and `asyncSerial` for name-based coroutine-level serial execution using a `ConcurrentHashMap` of `Mutex` instances.
- Documented non-reentrant deadlock risk when nesting same-name serial calls in `CoroutinesHolder` serial APIs.
- Upgraded Kotlin Gradle Plugin from 1.8.22 to 1.9.25.
- Upgraded kotlinx-coroutines-core and kotlinx-coroutines-android from 1.6.4 to 1.8.1.
- Added multilingual feature documentation for `AppEvent` (English, Chinese, Spanish).
- Updated `CoroutinesHolder` feature documentation with serial execution API usage and constraints.
- Restructured project and feature module documentation for consistent formatting and navigation across all supported languages.
- Added release note and changelog writing guidelines under `document/releases/`.
